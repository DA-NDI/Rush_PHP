    PHP - Web
    

    Le but était de lancer PAMP et de contrôler que l'on a un
serveur qui fonctionne. Avec plusieurs vhosts.

    

    

    ex00 - Dat vHost !
    

    

    Si PAMP n'est pas au moins en v3.0.0, vérifiez juste l'installation de 

    PAMP sur votre dump. Validez la question et passez à la suivante.

    

    ---

    

    Vérifiez les points suivants :

    - PAMP est installé et disponible sur le dump.

    - Le fichier 'D03.conf' est présent dans le dossier de rendu

    - Il contient la configuration d'un VirtualHost Apache écoutant sur le port 8100

    - Ce VirtualHost prend comme répertoire de base '$HOME/http/MyWebSite/d03'

    

    Pour tester cette configuration, faites un include de ce fichier tel que

    présenté sur le sujet et vérifier que tout fonctionne. Copier le rendu git

    dans le dossier cité dans la configuration ( vous pouvez faire un lien symbolique

    si le coeur vous en dit ).

    

    Les sous-répertoires ex01,ex02,etc...  présents sur le rendu git

    devront être accessibles sur l'URL et port du vhost, comme par

    exemple http://eXrXpX.42.fr:8100/ex01/phpinfo.php

    

    Si un de ces points ci-dessus n'est pas respecté, l'exercice est compté faux

    et vous passez à la suite

    

    

    

    

    

    





    

    

    

    

    

    

    ex01 phpinfo
    

    
Le but était de créer un fichier phpinfo.php qui execute et
affiche le résultat de la fonction phpinfo():

    

    

    ex01 phpinfo
    

    

    Ouvrir avec Chrome la page http://eXrXpX.42.fr:8xxx/ex01/phpinfo.php et 

    vérifier que cela affiche bien une page phpinfo()

    

    

    

    

    

    





    

    

    

    

    

    

    ex02 print_get
    

    
Le but était de créer une page print_get.php qui affiche toutes
les variables passées dans l'url, donc via la méthode GET

    

    

    ex02 print_get
    

    

    Run the following script:
 
    `` `  
 
    $> curl 'http://eXrXpX.42.fr:8xxx/ex02/print_get.php?dubo=tuveuxdupain&zaz=petitponey&6x7=42' 
 
    dubo: tuveuxdupain 
 
    zaz: petitponey 
 
    6x7: 42 
 
    $> 
 
    ` ``
 
    

    - The display is the same?

    - Also do your own tests

    

    

    

    

    

    





    

    

    

    

    

    

    ex03 cookie_crisp
    

    
The goal was to create a cookie_crisp.php page that allows you to create, read, and delete a cookie. Is that the case ?

    

    

    ex03 cookie_crisp
    

    

    - Create a cookie, it is necessary that the page returns well in the header a set-cookie. 

      Check with curl which displays the received HTTP header.   

      
     
        $> curl -v 'http://eXrXpX.42.fr:8xxx/ex03/cookie_crisp.php?action=set&name=boisson&value=white_russian' 
      
     

      The cookie is well transmitted?

    

    - Replay a created cookie. Do you have the same exit?

      
     
        $> curl -c cook.txt http://eXrXpX.42.fr:8xxx/ex03/cookie_crisp.php?action=set&name=dessert&value=tiramisu ' 
        $> curl -b cook.txt http: //eXrXpX.42. en: 8xxx / ex03 / cookie_crisp.php? action = get & name = dessert ' 
        tiramisu
      
    

    

    - Clear the cookie 'dessert' that we just created (only if the previous 2 tests are good of course).

      
     
        $> curl -c cook.txt http://eXrXpX.42.fr:8xxx/ex03/cookie_crisp.php?action=del&name=dessert ' 
        $> curl -c cook.txt http://eXrXpX.42.fr: 8xxx / ex03 / cookie_crisp.php? Action = get & name = dessert ' 
        $>
      
    

    

    

    

    

    

    





    

    

    

    

    

    

    ex04 raw_text

    ex04 raw_text

              $> curl -v http://eXrXpX.42.fr:8xxx/ex04/raw_text.php '

    ex06 members_only
    

    
The goal was to create a members.php page that requires a login / pass at the http protocol level. If the login is "zaz" and the pass "jaimelespetitsponeys" the answer must be a html page containing an img tag whose source is the image "/img/42.png" directly encoded in base64.

    

    

    ex06 members_only
    

    
Test without login or password that the page answers an
 
    error message and we receive the header "HTTP / 1.0 401
 
    Unauthorized"   

     
      $> curl -v http://eXrXpX.42.fr:8xxx/ex06/members_only.php 
      ... 
      <HTTP / 1.0 401 Unauthorized 
      ... 
      <WWW-Authenticate: Basic realm = "Member Space" 
      ... 
      < html> <body> 
        This field is only accessible to site members 
      </ body> </ html> $>

    we have the error message?

      <html> <body> 
        This area is only accessible to members of the site 
      </ body> </ html> 
      $ >
